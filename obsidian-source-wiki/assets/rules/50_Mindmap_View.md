# Mindmap View

## Goal

让结构清晰的 Markdown 文件可以稳定渲染成思维导图或大纲视图。

## Best Fit

最适合渲染成思维导图的是 `20_Knowledge` 文件。

`10_Sources` 可以结构化阅读，但通常不强求变成思维导图。

## Heading Rules

Knowledge 文件必须保持清晰标题层级：

```markdown
# 主标题

## 1. 一级模块

### 1.1 二级观点

### 1.2 二级观点

## 2. 一级模块

### 2.1 二级观点
```

不要跳级：

```markdown
# 标题
#### 直接四级标题
```

不要把多个主题塞进一个文件。

## Content Rules

- 每个 `##` 下面只讨论一个模块。
- 每个 `###` 下面只讨论一个具体观点。
- 列表可以用于并列要点。
- 表格只在确实需要比较时使用。
- 长段落要拆短，方便图形化阅读。

## Plugin Usage

思维导图插件读取标题和列表时，文件结构越规整，图越清楚。

因此 Knowledge 文件优先使用标题层级表达结构，不依赖复杂排版。
