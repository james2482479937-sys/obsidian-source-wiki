# Source Structuring

## Core Rule

Source is not a summary. Source is the readable, traceable version of source material.

Preserve source meaning and important wording. Improve heading structure, paragraph boundaries, obvious ASR/OCR noise, and navigation.

## Process

1. Read the full raw transcript, caption, or OCR first.
2. Identify the real topic blocks.
3. Replace mechanical headings such as `片段 1`, `图 3`, or `待语义整理片段`.
4. Use headings that describe the actual topic of that block.
5. Split long text into natural paragraphs.
6. Keep source links, raw links, platform, author, and capture references.
7. Add `Structure Tags`.
8. Leave distillation for Knowledge notes.

## Good Source Headings

Use headings like:

- `学生情况：专科背景，22 岁拿到字节高薪机会`
- `独立开发能力：面试官真正看重的是完整 AI 工具经验`
- `核心问题：模型没有记忆，但任务有连续性`
- `初始化动作：写功能清单、进度文件和启动脚本`
- `配置入口：供应商、协议和本地转发`

Avoid:

- `片段 1`
- `图 1`
- `求职建议 3`
- `视频转录`
- `OCR 内容`

## Source vs Knowledge

Source keeps the source readable.

Knowledge extracts durable ideas, definitions, methods, comparisons, and personal understanding.
